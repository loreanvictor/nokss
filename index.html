Enough CSS to get started


nokss is a drop-in stylesheet that works out of the box: there is no need for using special classes or attributes. In fact, no need to touch your HTML code at all. Instead, it relies on semantics of HTML itself to figure out how to style various elements, so the more semantic your HTML is, the more you get out of nokss.



Installation

Import in your CSS:

@import 'https://unpkg.com/nokss/dist/nokss.css';

Or link in your HTML:

<link rel="stylesheet"
  href="https://unpkg.com/nokss/dist/nokss.css" />


Usage

nokss works out of the box and styles elements based on their semantics rather than classes or attributes. You don't need to use any classes (like .btn or .container) to use its styles. Because nokss relies on HTML semantics for properly styling elements, the more senantic your HTML is, the more you'll get out of it. In the following section, the semantics that nokss looks for are described for each element / element type.


Buttons

Buttons are primary styled by default:

<button>Click me</button>

If buttons are grouped, only the last one is styled primary, the rest will have a secondary style:

<menu role="group">
  <button>Secondary</button>
  <button>Primary</button>
</menu>

Buttons in a toolbar have no primary and are styled differently:

<menu role="toolbar">
  <button>A</button>
  <button>B</button>
  <button>Longer</button>
</menu>
